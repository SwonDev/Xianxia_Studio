# Dev ↔ Prod Parity Contract

Goal: a change verified in `pnpm tauri:dev` is **guaranteed** to behave identically in the compiled installer. No more "worked in dev, broken in prod" surprises.

## The 4 historical divergences (now solved)

| # | Divergence (until v0.1.45) | Cure (v0.1.46+) |
|---|---|---|
| 1 | `sidecar-node` runs `dist/server.js` in BOTH dev and prod, but `dist/` was a stale snapshot whenever a `.ts` file was edited without rebuild. | `prepare-sidecars.mjs` now ALWAYS runs `tsc` (no conditional). `parity-check.mjs` fails if any `.ts` is newer than `dist/server.js`. |
| 2 | `resolve_sidecar` in Rust falls back to `<workspace>/apps/sidecar-py/` in dev when the runtime dir is empty. Different cwd, different relative paths. | `dev-sync.mjs` copies the staged sidecars into `<data_dir>/runtime/sidecar-{py,node}/` BEFORE `tauri dev` starts, with the version marker. From that moment, dev resolves sidecars at the prod path. |
| 3 | No automation between editing source and rebuilding the bundle. Easy to forget. | `pnpm tauri:dev` and `pnpm tauri:build` both invoke `dev-sync.mjs` / `parity-check.mjs` automatically. |
| 4 | No code-level smoke test catching invariant violations (e.g. `/g` regex missing, `intro_offset_seconds` not wired, hf_seed missing a repo). | `parity-check.mjs` enforces 9 invariants on every dev/build invocation. |

## Daily workflow

### Make a change
1. Edit the source you care about (any `.ts`, `.py`, `.rs`, prompt template, etc.).
2. Run `pnpm tauri:dev`. This:
   - Runs `sidecars:prepare` (recompiles TS, copies Python source).
   - Mirrors the staged sidecars into `<data_dir>/runtime/`, with version marker.
   - Runs `parity-check.mjs` — 9 invariants. If any fails, dev does NOT start; the failure includes a one-line remediation hint.
   - Starts `tauri dev`. The Tauri supervisor finds the sidecars at the prod path. Behaviour is identical to a real install.
3. Reproduce the issue / test the fix in the running app.

### Ready to ship
1. `pnpm tauri:build` — this also runs the parity check first. If it passes, the build proceeds. The bundled artifacts can be installed and they will behave EXACTLY like dev did, because both used the same `runtime/sidecar-*` tree generated by the same `prepare-sidecars` step.

### When iterating fast on a single sidecar
For a hot inner loop on Python (no need to restart Tauri):
```
# Terminal 1: tauri dev (already running)
# Terminal 2: just re-sync after each .py edit
pnpm dev:sync
# then restart the python sidecar via the in-app /diag controls
```
For a hot inner loop on Node:
```
pnpm dev:sync          # always-fresh dist/
```

## Invariants (mirror of `parity-check.mjs`)

1. `apps/sidecar-node/dist/server.js` mtime ≥ newest `apps/sidecar-node/src/*.ts` mtime.
2. `dist/render.js` contains the four `/__X__/g` global-regex replacements (catches the v0.1.42 thumbnail regression class).
3. `subtitles.py` exposes `intro_offset_seconds` so SRT/ASS can be shifted to match the 6 s intro card.
4. Rust pipeline (`pipeline/mod.rs`) sends `intro_offset_seconds: 6.0` to `/subtitles`.
5. `hf_seed.rs` lists `depth-anything/Depth-Anything-V2-small-hf` so the DepthFlow model gets seeded from the user's HF cache.
6. `depthflow.py` uses `subprocess.run(..., encoding="utf-8", errors="replace")` so DepthFlow's box-drawing progress chars don't crash on Spanish Windows (cp1252).
7. `/depthflow/batch` tolerates per-clip failure (`except HTTPException` + `output_path=""` placeholder) instead of aborting the whole batch.
8. `script.py` visual distillation uses Ollama `format=json` — required because Gemma 4B-abliterated returns empty body with `done_reason=length` on free-form prompts.
9. `tauri-build-local.mjs` uses `shell: true` in both spawnSync calls (Node 22+ refuses `.cmd` files with `shell:false` since CVE-2024-27980).

## When to add a new invariant

Whenever a regression is caused by a code-shape that can be detected by a one-line presence/absence check, add it to `parity-check.mjs`. The cost is ~5 lines of code; the payoff is never shipping that regression again.

Typical triggers:
- A bug where the source code was correct but the compiled artifact wasn't.
- A bug where dev didn't reproduce because of a path / env difference from prod.
- A "we already fixed this once" déjà vu.
